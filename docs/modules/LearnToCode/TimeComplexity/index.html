Time Complexity

Prerequisites: Nothing

Note: If you have never coded before, you may want to skip this until you have done a few simple problems.

Resources

Time Complexity is all about determining the worst-case scenario for the time it takes for code to run. This matters because probelms have a time limit for your code to run for each test-case which is usually a couple seconds. With good constant factors, this you are limited to around 5∗1085 * 10^85∗108 opperations for a problem.

Big O Notation

The way we measure a code's time complexity is in Big O Notation.

O(f(x))\mathcal{O}(f(x))O(f(x)) is how many opperations f(x)f(x)f(x) takes to complete in the worst-case-scenario.

Here are some example time complexities:

The code below runs in O(1)\mathcal{O}(1)O(1)since it does a constant number of opperations:

int a = 5; 
int b = 7; 
int c = a + b;

To keep it simple, we say that O(C)\mathcal{O}(C)O(C) becomes O(1)\mathcal{O}(1)O(1) as the extact number will not often affect the run-time.

The code below runs in O(n)\mathcal{O}(n)O(n) since it does a constant number of opperations nnn times:

for (int i = 1; i <= n; i++) {
  int a = 5; 
  int b = 7; 
  int c = a + b; 
}

Once again, we ignore CCC so O(n)\mathcal{O}(n)O(n) becomes O(n)\mathcal{O}(n)O(n).

If we add one more loop, it will run in O(n2)\mathcal{O}(n^2)O(n2).

for (int i = 1; i <= n; i++) {
  for(int j = 1; j <= n; j++) {
      int a = 5; 
      int b = 7; 
      int c = a + b; 
  }
}
