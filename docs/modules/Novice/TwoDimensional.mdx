We want to use the idea of prefix sums to solve the followimg problem, but if we did a prefix sum array for each row, 
it would take $\mathcal{O}(n)$ per query. 

If instead, we did a two dimensional prefix sum, where cell $(i, j)$ contains the sum of the elements $(x, y)$
where $x \leq i$ and $y \leq j$, we can answer the queries in $\mathcal(O)(1)$. 

The code below finds the sum of elemenst bounded by the rectange with a lower left point $(2, 1)$ and an upper right
point $(4, 3)$ (one-indexed)

<div style={{ transform: 'translateX(150px)' }}>
  <SyntaxHighlighter language="cpp" style={okaidia}>
{`vector<vector<int>> pre(n + 1, vector<int>(n + 1, 0)); 
for(int i = 0; i < n; i++) {
    for(int j = 0; j < n; j++) {
        pre[i + 1][j + 1] = a[i][j] + pre[i + 1][j] + pre[i][j + 1] - pre[i][j]; 
    }
}
cout << pre[4][3] - pre[1][3] - pre[4][0] + pre[1][0]; 
`}
  </SyntaxHighlighter>
</div>
